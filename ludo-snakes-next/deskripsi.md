<!-- DESKRIPSI LENGKAP PROJECT 
Project yang dikembangkan adalah permainan papan berbasis web yang menggabungkan mekanik Ludo dan ular tangga dalam satu sistem terpadu. Permainan menggunakan papan dengan total 50 kotak dan dapat dimainkan oleh maksimal empat pemain. Setiap pemain hanya memiliki satu pion yang harus digerakkan dari titik awal hingga mencapai garis akhir. Pergerakan pion ditentukan oleh tiga dadu, yaitu dua dadu putih yang dijumlahkan dan satu dadu merah yang berfungsi sebagai pengurang hasil tersebut. Jika hasil pengurangan bernilai negatif, maka pion akan bergerak mundur sesuai nilai tersebut. Untuk dapat mengeluarkan pion dari base ke titik start, pemain harus mendapatkan total angka enam dari dua dadu putih tanpa melibatkan dadu merah. Papan permainan juga memiliki elemen tambahan berupa tangga yang menaikkan posisi dan ular yang menurunkan posisi secara acak. Selain itu, terdapat satu kotak event spesial di posisi 25 yang memberikan efek acak antara langsung menuju finish atau kembali ke start.
Permainan juga memiliki mekanik interaksi antar pemain yang memungkinkan terjadinya tabrakan atau capture. Jika dua pemain berada pada kotak yang sama, maka pion yang lebih dulu berada di posisi tersebut akan dikembalikan ke titik start. Sistem permainan bersifat turn-based sehingga setiap pemain hanya dapat melakukan aksi saat gilirannya tiba. Validasi giliran menjadi bagian penting untuk memastikan tidak terjadi konflik aksi antar pemain. Seluruh perhitungan logika permainan dilakukan di sisi server untuk menjaga konsistensi data. Game state disimpan dalam bentuk terstruktur agar dapat digunakan kembali selama permainan berlangsung. Struktur ini mencakup posisi pemain, giliran aktif, serta elemen papan seperti ular dan tangga. Dengan pendekatan ini, sistem mampu menjaga integritas permainan secara menyeluruh.
Arsitektur sistem menggunakan pendekatan full-stack dengan Laravel sebagai backend utama yang menangani API, autentikasi, dan komunikasi dengan database MySQL. Data dikirim dan diterima dalam format JSON untuk mempermudah integrasi antar sistem. Sistem juga menggunakan Node.js untuk menangani komunikasi real-time melalui WebSocket. Teknologi ini memungkinkan sinkronisasi data antar pemain tanpa perlu melakukan refresh halaman. Autentikasi pengguna dilakukan menggunakan Google OAuth sehingga setiap pemain memiliki akun dan identitas unik. Sistem dijalankan menggunakan server lokal berbasis Apache HTTP Server. Untuk kebutuhan akses publik selama pengembangan, digunakan ngrok sebagai jembatan ke internet. Kombinasi teknologi ini memastikan sistem dapat berjalan secara stabil dan scalable.
Sistem multiplayer dirancang menggunakan konsep room dan invite yang sederhana namun efektif. Pemain dapat membuat room dan mengundang pemain lain dengan memasukkan user_id tanpa perlu fitur daftar pemain online. Ketika undangan diterima, pemain akan bergabung dalam room yang sama dan menunggu permainan dimulai. Setiap room memiliki informasi seperti status permainan, daftar pemain, dan giliran aktif. Database dirancang dengan beberapa tabel utama seperti users, invites, rooms, room_players, dan game_states. Komunikasi real-time antar pemain dilakukan melalui WebSocket dengan berbagai event seperti join_room, roll_dice, move_player, dan next_turn. Semua aksi pemain divalidasi di server untuk menghindari kecurangan. Pendekatan ini memastikan permainan berjalan sinkron di semua perangkat pemain.
Antarmuka pengguna dirancang sederhana dan fokus pada pengalaman bermain. Halaman login menyediakan autentikasi dan mengarahkan pengguna ke dashboard setelah berhasil masuk. Dashboard hanya memiliki satu fungsi utama yaitu membuat room dan mengundang pemain menggunakan user_id. Halaman room menampilkan informasi pemain dan status sebelum permainan dimulai. Pada halaman gameplay, papan permainan ditampilkan di tengah dengan posisi pemain di sekitar papan secara dinamis sesuai perspektif masing-masing. Mekanik dadu ditampilkan dalam bentuk slot machine dengan tiga kolom yang berputar dari atas ke bawah. Tombol spin hanya aktif saat giliran pemain dan tetap terlihat oleh semua pemain. Hasil permainan ditampilkan secara sederhana dengan fokus pada pemenang tanpa informasi tambahan lainnya.
 
DETAILNYA
Permainan yang dikembangkan merupakan game papan berbasis web yang menggabungkan mekanik Ludo dan ular tangga dengan total 50 kotak dan maksimal 4 pemain, di mana setiap pemain hanya memiliki satu pion. Pergerakan pion ditentukan oleh tiga dadu, yaitu dua dadu putih yang dijumlahkan dan satu dadu merah yang mengurangi hasil tersebut sehingga langkah bisa maju atau mundur. Untuk memulai permainan, pemain harus mendapatkan total angka 6 dari dua dadu putih agar dapat mengeluarkan pion ke titik start, dan dadu merah tidak berlaku pada kondisi ini. Papan permainan memiliki elemen tambahan berupa tangga dan ular dengan posisi acak, serta satu kotak event spesial di posisi 25 yang memberikan efek acak antara langsung ke finish atau kembali ke start. Selain itu, terdapat mekanik interaksi antar pemain di mana jika satu pion berhenti di kotak yang sama dengan pion lawan, maka pion yang lebih dulu berada di kotak tersebut akan dikembalikan ke titik start.
Versi final ambisius dari project ini menggunakan arsitektur full-stack dengan Laravel sebagai backend utama untuk REST API, autentikasi, dan pengelolaan database MySQL. Sistem juga memanfaatkan Node.js untuk menangani komunikasi real-time menggunakan WebSocket agar gameplay dapat berlangsung secara interaktif antar pemain. Pertukaran data antar sistem menggunakan format JSON, serta seluruh operasi data dikelola melalui endpoint API dengan konsep CRUD yang terstruktur. Autentikasi pengguna dilakukan menggunakan Google OAuth sehingga setiap pemain memiliki akun unik dan ID untuk sistem undangan multiplayer. Aplikasi dijalankan di server lokal menggunakan Apache HTTP Server dan diekspos ke internet menggunakan ngrok untuk kebutuhan pengujian dan integrasi layanan eksternal.
Sistem multiplayer game ini dirancang menggunakan konsep room dan invite, di mana setiap pemain yang telah login melalui Google OAuth atau metode autentikasi lainnya dapat mengirim undangan kepada pemain lain untuk bergabung dalam satu sesi permainan. Setiap pemain memiliki identitas unik berupa user_id yang digunakan untuk proses pengelolaan data dan interaksi dalam sistem. Ketika undangan diterima, sistem akan membuat sebuah room dengan room_id sebagai identitas utama pertandingan. Setiap room menyimpan informasi penting seperti status permainan, jumlah pemain, giliran aktif, serta kondisi permainan secara keseluruhan. Data kondisi permainan atau game state disimpan dalam format JSON agar fleksibel dalam menyimpan posisi pion, elemen ular dan tangga, serta event khusus. Struktur database menggunakan MySQL dengan tabel utama seperti users, invites, rooms, room_players, dan game_states untuk memastikan pengelolaan data yang terorganisir. Komunikasi antar pemain dilakukan secara real-time menggunakan WebSocket yang dijalankan dengan Node.js. Sistem WebSocket menangani berbagai event penting seperti join_room, roll_dice, move_player, capture, next_turn, dan game_end selama permainan berlangsung. Seluruh proses logika permainan, termasuk perhitungan langkah dari dadu, validasi giliran, serta interaksi antar pemain dilakukan di sisi server untuk menjaga konsistensi data. Pendekatan ini memastikan sinkronisasi antar perangkat pemain tetap terjaga serta meminimalkan potensi kecurangan dalam permainan online.
Sistem antarmuka game ini dimulai dari halaman login menggunakan Google OAuth atau metode lain yang langsung mengarahkan pengguna ke dashboard setelah berhasil masuk. Pada dashboard, pengguna hanya memiliki satu opsi utama yaitu membuat room, kemudian mengundang pemain lain dengan memasukkan user_id tanpa adanya daftar pemain online atau fitur sosial tambahan. Sistem invite bekerja melalui notifikasi yang dapat diterima atau ditolak, dan jika diterima maka pemain akan masuk ke room yang sama untuk memulai permainan. Halaman room menampilkan informasi dasar seperti room ID, daftar pemain, serta status permainan sebelum dimulai oleh host. Pada halaman gameplay, board game ditempatkan di tengah dengan posisi profil pemain yang tersebar di sekitar board dan bersifat dinamis sesuai perspektif masing-masing pemain. Mekanik dadu ditampilkan dalam bentuk slot machine dengan tiga kolom yang berputar dari atas ke bawah dan dikontrol melalui tombol spin yang hanya aktif saat giliran pemain. Seluruh interaksi termasuk animasi dadu dan pergerakan pion disinkronkan secara real-time menggunakan WebSocket berbasis Node.js, dan hasil akhir permainan ditampilkan secara sederhana dengan fokus pada pemenang tanpa informasi tambahan lainnya.
